🔹 Лаба 2 — как объяснять (с ориентирами)
1. Общая логика

Система показывает, как происходит анализ репутации бренда.

👉 Слева — входные данные
👉 Справа — результат
👉 Сверху — правила(управление)
👉 Снизу — ресурсы(механизмы)

2. IDEF0 (0 уровень)

👉 В центре:
система анализа репутации

👉 Слева (вход):

отзывы о бренде
данные о бренде

👉 Сверху (управление):

методика оценки
правила анализа

👉 Снизу (механизмы):

база данных
пользователь

👉 Справа (выход):

оценка репутации
аналитический отчёт
3. IDEF0 (1 уровень)

👉 Система разбита на этапы

👉 Слева → направо:

сбор отзывов
анализ отзывов
формирование отчётов

👉 Поток:
слева → направо

4. IDEF0 (2 уровень)

👉 Детализация процессов

👉 Например:

сначала подготовка отзывов
потом анализ
потом расчёт репутации

👉 То есть показывается, как именно работает каждый этап

5. DFD (0 уровень)

👉 Слева:
источники отзывов

👉 Справа:
пользователь

👉 В центре:
система

👉 Потоки:

слева → в систему (отзывы)
справа → в систему (запрос)
из системы → вправо (результат)
6. DFD (1 уровень)

👉 Сверху слева:
база данных

👉 В центре:
обработка

👉 Справа:
анализ

👉 Поток:

БД → обработка → анализ → пользователь

👉 И дополнительно:

анализ → БД
(сохранение результатов)

7. Что говорить в целом

Система получает отзывы о бренде,
обрабатывает их и анализирует.
Далее рассчитывается репутация,
и результат передаётся пользователю в виде отчёта.

🔥 Короткая версия (если быстро отвечать)

Слева — входные данные, справа — результат.
Сверху — правила работы, снизу — ресурсы системы.
В IDEF0 показаны функции системы,
в DFD — движение данных.
Данные проходят путь: сбор → анализ → результат.
