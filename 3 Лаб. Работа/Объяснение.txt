Лаба 3 — как объяснять (с ориентирами)
1. Общая логика

Схема делится на 2 части:

👉 Сверху — сервер
👉 Снизу — пользователи

2. Сервер (верхняя часть)

👉 Слева внутри сервера:

БД отзывов
обработка отзывов

👉 Справа внутри сервера:

модуль анализа

👉 Поток внутри сервера:
слева → направо

БД → обработка → анализ
3. Сеть (справа)

👉 Справа от сервера:

подключение
брандмауэр
HTTP
интернет
4. Пользователи (снизу)

👉 Снизу:

рабочие места аналитиков

👉 Поток:
снизу → вправо → вверх

пользователь → интернет → сервер
5. Что говорить в целом

Пользователь снизу отправляет запрос через интернет.
Запрос проходит через брандмауэр и попадает на сервер.
На сервере данные обрабатываются и анализируются.
Результат возвращается пользователю.

🔥 Короткая версия (если быстро отвечать)

Слева — входные данные, справа — результат.
Сверху — правила работы, снизу — ресурсы системы.
В IT-схеме сверху сервер, снизу пользователи, справа сеть.
Данные идут от пользователя через интернет к серверу и обратно.
